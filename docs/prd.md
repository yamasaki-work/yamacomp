# PRD：YamaComp（仮）— レベル別登山提案 + 装備チェッカー

## 1. プロダクト概要
ユーザーの体力レベル・条件を入力すると、AIが適切な山を提案し、
必要な装備リストを自動生成するWebアプリ。
「次どこ登ろう」と「何持っていけばいい」を一度に解決する。

## 2. 解決する課題
- 自分の体力に合う山をどう選べばいいかわからない
- 登山ごとに装備を一から調べ直すのが面倒
- 季節・エリア・目的（景色・観光・難易度）で条件を絞り込めるサービスがない
- YAMAPは「行く山が決まってから使う」ツール。「山を決める前」のツールがない

## 3. ターゲットユーザー
- 年5回程度、春〜秋に登山する初心者〜中級者
- 車・電車どちらでもアクセスする
- 景色・観光・難易度など目的がそのつど変わる

## 4. ユーザーインプット（入力項目）

### 体力レベル
普段の運動習慣・階段での息切れ具合・歩行時間などの質問に答えてレベルを診断

### アクセス条件
エリア（関東・関西・中部など）、車での所要時間 or 電車での所要時間、日帰り or 1泊2日

### 登山スタイル
目的（景色重視 / 周辺観光込み / 難易度チャレンジ / ゆるハイク）
季節のおすすめ優先か否か

### 歩行時間
希望の歩行時間（〜3時間 / 3〜5時間 / 5〜7時間 / 7時間以上）

## 5. アウトプット
- 条件に合う山の提案（3〜5件）+ 各山の概要・難易度・特徴・アクセス
- 選んだ山に対応した装備チェックリスト（季節・難易度・泊まりか否かで動的生成）
- 免責文：「本アプリの情報は参考用です。実際の登山計画はYAMAP等と組み合わせてください」

## 6. 技術スタック
- フロントエンド：HTML / CSS / JavaScript（Claude Artifacts）
- AI：Claude API（Anthropic）
- データ：山・装備データをMDファイルで管理しClaudeに読み込ませる
- デプロイ：GitHub Pages
- バージョン管理：Git / GitHub

## 7. 成功指標（MVP）
- 入力から山の提案が30秒以内に表示される
- 装備リストが選んだ山・季節・スタイルに応じて変わる
- 自分が次の登山計画で実際に使える状態になっている

## 8. スコープ外（v2以降）

- 入山者数データの連携
- ユーザーアカウント・登山履歴の保存
