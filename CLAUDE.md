# CLAUDE.md — YamaComp プロジェクト文脈ファイル

## プロジェクト概要
レベル別登山提案 + 装備チェッカーWebアプリ。
ユーザーの体力・条件を入力すると、AIが山を提案し装備リストを生成する。
詳細はdocs/prd.mdを参照。

## 技術スタック
- フロントエンド：HTML / CSS / JavaScript
- AI：Claude API（Anthropic）
- データ管理：MDファイル（山データ・装備データ）
- デプロイ：GitHub Pages
- バージョン管理：Git / GitHub

## コーディング規約
- コメントはすべて日本語で書く
- 変数名・関数名は英語のキャメルケース（例：mountainLevel, equipmentList）
- エラーは必ずconsole.errorで出力し、ユーザーには日本語メッセージを表示
- APIキーは環境変数で管理。コードにハードコードしない
- ファイル構成：index.html / style.css / main.js / data/mountains.md / data/equipment.md

## AIへの指示スタイル
- 山の提案は必ず3〜5件。各提案に「山名・難易度・歩行時間・特徴・アクセス概要」を含める
- 装備リストは季節・難易度・日帰り泊まりで動的に変える
- 免責文を必ずUIに表示：「本アプリの情報は参考用です。実際の登山計画はYAMAP等と組み合わせてください」
- 安全に関わる情報（天気・難所）は「必ず現地情報を確認してください」と添える

## データファイルの場所
- `data/mountains.md`：山の参照データ（新規追加時のマスター）
- `data/equipment.md`：装備データ（季節・難易度・泊まり別の装備リスト）
- **実際の山DBは `main.js` の `MOUNTAINS_DB` 配列に直接定義されている**。`data/mountains.md` を更新しても自動でアプリに反映されない。両者が乖離しないよう注意すること
- 新しい山を追加するときは `/add-mountain` スキルで `data/mountains.md` に追記し、その後 `main.js` の `MOUNTAINS_DB` にも同じ内容を追加する

## 重要な制約
- リアルタイムのクマ情報・入山者数はスコープ外（v2以降）
- 外部APIは使わない（Claude API以外）
- 山データは手動でMDに追加していく方式
- YAMAPやヤマレコのデータをスクレイピングしない

## 入力フロー（確定設計）
4ステップ（体力診断→アクセス→スタイル→歩行時間）を廃止し、**2ステップに整理済み**。
体力診断3問を削除した理由：ユーザーの自然な思考順（日帰り→エリア→コースタイム→目的）と合わず、離脱を招くため。

- **Step 1「今回のプラン」**：日程（日帰り/1泊2日）・出発エリア・移動手段
- **Step 2「どんな山にしますか？」**：希望コースタイム（直接選択）・楽しみたいこと（複数選択）

コースタイムから難易度上限を推定するロジック（`getMockMountainSuggestions`内）：
- 〜3時間 → difficultyLevel 1（初級）まで
- 3〜5時間 → 2（中級）まで
- 5〜7時間以上 → 3（上級）まで

## 実装済み機能
- **山カードグリッド**：2列（モバイル480px以下は1列）。カードにSVG生成の山イラスト・難易度バッジ・順位バッジ付き
- **比較機能**：最大3件まで選択 → 画面下にスティッキーバー → 「比較する」でモーダル表示
- **URLシェア**：入力条件＋山名リストをBase64エンコードして `#share=<hash>` にURL埋め込み。ページロード時に自動復元する（`initShareFromUrl`）

## デザインシステム
`style.css` はCSS変数（`:root`）でカラー・シャドウ・ボーダーラジウス・トランジションをトークン化済み。
- ブランドカラー：`--c-500`（山の緑 `#368558`）
- アクセント：`--ac-500`（夕焼けオレンジ `#e06236`）
- デザインを変えるときはまず `:root` のトークンを確認すること。ハードコードの色は使わない

## 利用可能なスキル

### `/add-mountain`
`data/mountains.md` に新しい山エントリを追加する。

**呼び出し例：**
```
/add-mountain 山名：○○山、エリア：関東、難易度：中級、歩行時間：4〜6時間（往復）、特徴：〜〜、アクセス（電車）：〜〜、アクセス（車）：〜〜
```

**渡す情報：** 山名・エリア・難易度・歩行時間・特徴（100〜150字）・電車アクセス・車アクセス。不明な項目は省略してよい（「要確認」と記入される）。

**注意：** スキルは `data/mountains.md` のみ更新する。アプリに反映するには `main.js` の `MOUNTAINS_DB` にも同じ山を追加すること。
